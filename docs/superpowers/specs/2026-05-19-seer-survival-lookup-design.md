# SEER 口腔/咽部癌症生存查表网站设计

## 目标

构建一个科研展示版网站：用户输入性别、肿瘤部位、组织学大类、年龄、T/N/M 分期后，网站基于当前 SEER 导出表中相似病例组，展示总体生存估计结果。

网站运行时必须使用查表法。所有生存统计都在离线阶段预先计算，网站只读取预生成的查找表，不在后台实时训练模型、调用模型推理或使用 AI 推理。

## 非目标

- 不用于临床决策。
- 不预测个体真实寿命。
- 不做癌症特异生存，只做总体生存。
- 不在网站运行时执行 Cox、机器学习模型或其他实时预测算法。
- 第一版不纳入治疗变量、合并症、吸烟饮酒、ECOG、复发信息等表中缺失或不稳定变量。

## 数据来源与结局定义

数据文件为 `统计表.xlsx`，`Sheet1` 范围为 `A1:TY40867`，包含 40,866 条病例记录和 545 个字段。

总体生存结局：

- 时间：`Survival months`
- 事件状态：`Vital status recode (study cutoff used)`
- `Dead` 记为事件发生
- `Alive` 记为右删失

所有输出均解释为“SEER 历史病例中相似病例组的总体生存分布”，不是个体确定性预测。

## 输入变量

第一版输入变量如下：

| 网站输入 | 源字段 | 处理方式 |
|---|---|---|
| 性别 | `Sex` | 保留 Male/Female |
| 年龄 | `Age at diagnosis` | 用户输入连续年龄，运行时映射到年龄组 |
| 肿瘤部位 | `Site recode ICD-O-3/WHO 2008` | 下拉选择 |
| 组织学大类 | `Histology recode - broad groupings` | 下拉选择 |
| T 分期 | 派生统一 T 字段 | 下拉选择 |
| N 分期 | 派生统一 N 字段 | 下拉选择 |
| M 分期 | 派生统一 M 字段 | 下拉选择 |

年龄查表分组第一版采用：

- `<40`
- `40-49`
- `50-59`
- `60-69`
- `70-79`
- `80+`

用户仍输入具体年龄，例如 `63`，前端映射到 `60-69` 后查表。

## TNM 统一规则

离线预处理生成统一的 `t_stage`、`n_stage`、`m_stage`：

- 2010-2015 年病例优先使用 `Derived AJCC T/N/M, 7th ed (2010-2015)`
- 2016 年病例优先使用 `Derived SEER Combined T/N/M (2016+)`
- 标准化为：
  - T：`T0/T1/T2/T3/T4/TX/Unknown`
  - N：`N0/N1/N2/N3/NX/Unknown`
  - M：`M0/M1/MX/Unknown`
- 子分期合并到主分期，例如 `T2a`、`T2b` 合并为 `T2`，`N2a`、`N2b`、`N2c` 合并为 `N2`
- `Blank(s)`、空值、不适用和无法解析值统一为 `Unknown`

## 查找表生成

离线脚本按以下组合分层：

```text
sex
site
histology_group
age_group
t_stage
n_stage
m_stage
```

每个组合使用 Kaplan-Meier 方法计算总体生存分布，并生成一行查找结果：

```text
sex
site
histology_group
age_group
t_stage
n_stage
m_stage
sample_size
event_count
censor_count
median_survival_months
survival_12m
survival_36m
survival_60m
curve_months
curve_survival_probs
matching_level
data_quality_flag
```

`curve_months` 和 `curve_survival_probs` 存为数组，用于前端绘制生存曲线。

## 样本量与回退规则

查表法的核心风险是组合过细导致样本量不足。因此结果必须带样本量标记。

样本量规则：

- `sample_size >= 50`：直接显示完整组合结果
- `20 <= sample_size < 50`：显示结果，同时标记“样本量较小”
- `sample_size < 20`：不显示该细分组合结果，自动回退到更粗组合

回退顺序：

1. 完整组合：性别 + 部位 + 组织学大类 + 年龄组 + T + N + M
2. 去掉性别
3. 去掉组织学大类
4. 年龄组合并为更粗年龄组：`<60`、`60-69`、`70+`
5. 只保留部位 + T + N + M
6. 保留部位 + M 分期
7. 保留部位总体结果

前端必须显示实际使用的匹配层级。例如：

> 当前结果基于较粗匹配条件：肿瘤部位 + T/N/M 分期。完整组合样本量不足。

## 网站输出

预测结果页面显示：

- 匹配条件
- 实际匹配层级
- 样本量、事件数、删失数
- 中位总体生存期
- 1 年总体生存率
- 3 年总体生存率
- 5 年总体生存率
- Kaplan-Meier 生存曲线
- 数据质量提示

中位生存期规则：

- 如果曲线下降到 50% 以下，显示中位生存月数
- 如果随访期内曲线未降到 50%，显示“中位生存期未达到”

推荐措辞：

> 基于 SEER 历史数据中相似病例组，估计中位总体生存期为 X 个月。

禁止措辞：

> 你还能活 X 个月。

## 页面结构

第一版建议做成静态或近静态科研展示网站：

- 预测页：输入表单、结果卡片、生存曲线、匹配说明
- 方法页：数据来源、变量定义、KM 查表方法、样本量回退规则
- 模型表现页：总体样本量、主要分组分布、查表覆盖率、低样本量组合比例
- 限制说明页：非临床用途、SEER 数据限制、缺失变量、总体生存定义

运行时依赖：

- `survival_lookup.json`
- `options.json`
- 前端图表库

不需要运行时数据库和模型服务。

## 验证与质量检查

离线生成查找表后，需要验证：

- 总记录数与源表一致
- `Dead`/`Alive` 编码正确
- 生存月数为非负数
- TNM 标准化后的类别分布合理
- 每个查表组合的样本量、事件数、删失数可追溯
- 1/3/5 年生存率在 0 到 1 之间
- KM 曲线单调不增
- 中位生存期计算逻辑正确
- 回退层级能覆盖所有允许输入组合

展示性指标：

- 总样本量
- 可直接完整匹配的组合比例
- 需要回退的组合比例
- 样本量不足的组合比例
- 不同肿瘤部位的病例数分布

## 风险与边界

主要风险：

- 组合过细导致样本量小，需要清晰回退
- SEER 缺少吸烟、饮酒、合并症、体能状态、详细治疗方案等重要预后变量
- 2010-2015 与 2016 的 TNM 字段来源不同，统一规则必须透明
- 总体生存受非癌症死亡影响，不等于癌症特异死亡风险

页面必须明确：

> 本工具仅用于科研展示和方法演示，不用于临床诊断、治疗选择或个体预后判断。
